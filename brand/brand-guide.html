01 — Foundation

Design Philosophy

Dormy's visual identity is built around one idea: editorial warmth. The confidence of a premium publication. The approachability of a trusted partner.

📖
Editorial Warmth
Inspired by high-quality editorial design. Generous whitespace, serif headings, and warm cream tones create a feeling of considered quality — not startup hustle.
🎯
Specific Over Generic
Every visual element should feel intentional. Coral is not "red." Cream is not "white." The specificity of the palette signals that this product was designed, not assembled.
🤝
Partner, Not Tool
The warmth of the color palette and the humanity of the serif type communicate that Dormy has a point of view — and is on the founder's side.
⚡
Restrained Motion
Animations are subtle and purposeful. No gratuitous effects. Every transition should feel like a considered editorial choice, not a demonstration of technical capability.

07 — Rules

Do's & Don'ts

Common mistakes and their corrections. These rules protect the brand from dilution and keep the visual system coherent.

Color

✓ Do
  • Use Cream (#FDFCFA) as the page background — not pure white
  • Use Coral sparingly for CTA buttons, accents, and key highlights
  • Pair navy text on cream/white backgrounds for body copy
  • Use Warm Gray (#888580) for secondary and caption text
  • Use the border color (#E8E6E1) for all dividers and card borders
✗ Don't
  • Don't use pure white (#FFFFFF) as a page background
  • Don't use Coral for large background fills — it's an accent, not a canvas
  • Don't use generic blue for links — use Coral or Navy
  • Don't mix in off-brand accent colors (purple, green, etc.) without approval
  • Don't use black (#000000) — use Navy for maximum contrast

Typography

✓ Do
  • Use DM Serif Display for all H1, H2, H3 headings
  • Use italic DM Serif Display for accent phrases in hero copy
  • Use DM Sans at 600 weight for buttons, nav, and UI labels
  • Use generous letter-spacing (2px) for ALL-CAPS labels and eyebrows
  • Maintain hierarchy: Display → H1 → H2 → Body → Caption
✗ Don't
  • Don't use Outfit for body text or UI copy — brand/logo only
  • Don't use bold (700+) weight in DM Serif Display
  • Don't use ALL CAPS for body text — only micro labels
  • Don't mix DM Serif Display and DM Sans in the same heading
  • Don't go below 11px for any visible text

Logo

✓ Do
  • Use the SVG version whenever possible (scalable, sharper)
  • Match the logo variant to the background color
  • Maintain clear space of at least 1× D-height on all sides
  • Use the wordmark as the primary identifier (not just the D mark)
  • Use the D mark only for favicons, app icons, and small-format uses
✗ Don't
  • Don't stretch, skew, or rotate the logo
  • Don't place the logo on busy or low-contrast backgrounds
  • Don't recreate the wordmark using Outfit text — always use the SVG file
  • Don't add drop shadows or effects to the logo
  • Don't use logos from archive/ — v5 only

Voice & Copy

✓ Do
  • Write in first person plural ("we") — Dormy is a partner, not a product
  • Lead with specific outcomes, not features
  • Acknowledge that fundraising and GTM are genuinely hard
  • Use concrete numbers and specifics when available
  • End CTAs with directional signals ("→", "Get Started →")
✗ Don't
  • Don't lead with "AI-powered" — it's table stakes, not a differentiator
  • Don't use "unlock," "supercharge," or "10x"
  • Don't write in third person ("Dormy helps you...") — use "We"
  • Don't use all-caps headlines in body copy
  • Don't over-explain — respect the founder's intelligence
